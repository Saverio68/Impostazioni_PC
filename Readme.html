Note sui parametri chiave:

  • input: ["text"] per testuale, ["text", "image"] per vision (LLaMA3.2, Qwen3-VL, GLM-OCR)
  • reasoning: true per modelli con reasoning attivo (Qwen3, DeepSeek R1, Ornith)
  • num_ctx: context window dinamica per ogni modello, mai inferiore a 32K dove possibile
  • temperature: 0.2 per coding (consistente e deterministico), 0.5 per vision (più flessibile)
  • top_p: 0.9 standard per tutti i modelli — bilanciamento tra diversità e coerenza
  • maxTokens: allineato al context window o a limiti ragionevoli specifici del modello, garantendo output lunghi senza truncature premature
  • num_predict: pari a maxTokens per evitare truncatura anticipata

🔧 Note sulle scelte di configurazione

Perché questi parametri per coding sessions lunghe?

  • num_ctx=131072: Context window massimizzata dove possibile (48 GB RAM permettono ampi context senza swap)
  • temperature=0.2: Bassa temperatura per output consistenti e replicabili — cruciale per coding
  • temperature=0.5 per vision: maggiore flessibilità nell'interpretazione di immagini/grafici
  • n. predicit = maxTokens: Nessun limite artificioso all'output — i modelli possono generare quanto necessario
  • use_mlock=false: macOS deve condividere la memoria tra process; il blocco VRAM causerebbe OOM su 48GB
  • release_after_inference_delay=60s: I modelli rimangono in VRAM per 1 minuto — riduce il tempo di loading quando si switcha velocemente tra progetti
